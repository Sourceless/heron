Heron

A unified, immutable infrastructure database. It continuously ingests the state of your infrastructure from cloud providers and developer platforms, stores every fact with a timestamp, and makes the entire corpus queryable with Datalog.

The Problem

The Visibility Gap

There is no single place to ask "what is my infrastructure right now?" AWS Console shows EC2. GitHub shows repositories. A platform engineer investigating an incident must context-switch between five consoles, three CLIs, and a pile of Terraform state files.

Compliance Drift

Compliance goals are written in documentation but enforced nowhere. Violations accumulate silently between audits. By the time a security engineer runs a manual check, dozens of resources may be non-compliant — with no record of when they drifted or who changed them.

Shift-Left Failure

Even teams that have adopted Infrastructure-as-Code struggle to enforce policy at plan time. Terraform plan output is text, not data. Pre-flight checks require bespoke parsing of HCL or JSON, are brittle, and cannot reason about the effect of a change on existing state.

The Solution

Heron is an immutable infrastructure database built on Datomic. Every fact about every resource is stored with a timestamp and never overwritten — history is a first-class feature, not an afterthought. Compliance goals are expressed as Datalog queries, not scripts: a check passes when its query returns zero results. Pre-flight enforcement is possible because Heron can project a Terraform plan onto existing state and evaluate compliance before anything is applied.

Compliance as a Query

Checks are just Datalog queries. If the query returns zero results, the check passes. Here is the built-in check that enforces S3 public-access blocking across every bucket in your account:

Datalog — :s3/public-access-blocked
[:find ?name ?region
 :where
 [?e :aws.s3.bucket/name ?name]
 [?e :aws.s3.bucket/region ?region]
 [?e :aws.s3.bucket/public-access-blocked false]]

No scripts, no SDK calls, no bespoke parsers — just a declarative query evaluated against the full historical corpus of your infrastructure.

Roadmap

No time estimates — this is a solo project sequenced by dependency and risk.

Phase 0 — Foundation In progress

Prove the core data model. Datomic schema, IConnector protocol, S3 and IAM connectors, sync engine, REPL-based Datalog queries.

Phase 1 — MVP Core Planned

All MVP features working: full AWS + GitHub connectors, check and report engines, HTTP API, CLI, Chime scheduler.

Phase 2 — Operationalization Planned

Reliable for daily use: multi-account AWS, health API, schema discovery, custom check loading, beta release.

Phase 3 — Real-Time Connectors Planned

Reduce data latency to <60 seconds via EventBridge and GitHub webhooks; polling as fallback.

Phase 4 — Strong Checks Planned

Pre-flight compliance enforcement in CI/CD: Terraform plan projection via datomic.api/with, GitHub Action integration.

Phase 5 — Platform Expansion Planned

Connector SDK, GCP and Azure connectors, browser UI, multi-tenancy, RBAC.
